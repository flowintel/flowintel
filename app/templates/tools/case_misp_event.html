How a case is created from a MISP event
  • The case is created from the chosen template, with the title you provided, marked as private and linked to the MISP instance so it can be re-synced later.
  • Description is set to the MISP event info; ticket id is set to MISP Event: <event id>.
  • Tags and galaxy clusters that already exist in Flowintel are attached to the case; the rest are ignored.
  • Only the MISP objects you ticked are converted into Flowintel MISP objects (their attributes come along).
  • The four switches above optionally add a task with an event-info note, a task with a markdown table of the (selected) individual attributes, the MISP event reports as .md attachments, and the files attached to the MISP event (attachment / malware-sample attributes).
